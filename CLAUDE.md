# VKR Checker — Сервис автоматической проверки магистерских работ

## Описание проекта
Веб-сервис для автоматической проверки ВКР (магистерских диссертаций и проектов) по чек-листу. Студент загружает документ (.docx/.pdf), указывает ссылку на базу данных (Яндекс.Диск) и презентацию — система анализирует работу через OpenAI GPT и выдаёт подробный отчёт.

Проект разработан для образовательной программы «Интегрированные коммуникации» НИУ ВШЭ.

## Технологический стек
- **Frontend + Backend:** Next.js 14 (App Router), React, TypeScript
- **Стили:** Tailwind CSS
- **БД:** SQLite (через better-sqlite3), файл `data/vkr.db`
- **AI:** OpenAI API (модель задаётся в `.env`, по умолчанию `gpt-4o-mini`)
- **Парсинг документов:** mammoth (docx), pdf-parse (pdf), xlsx (Excel)
- **Интеграция:** Яндекс.Диск API (публичные ссылки, без OAuth)

## Ключевые архитектурные решения

### Чек-лист
- Пункты генерируются динамически в `checklist.ts` в зависимости от типа работы (проект/диссертация) и выбранных методов исследования (интервью, фокус-группы, опрос и т.д.)
- Каждый пункт имеет `auto: true/false` — автоматическая проверка через GPT или ручная
- Пункты длительности аудио и качества звука всегда `auto: false` (невозможно проверить программно)

### Проверка базы данных
- Яндекс.Диск API используется для получения списка файлов по публичной ссылке
- Подпапки обходятся рекурсивно (1 уровень вложенности)
- Файлы .xlsx/.csv парсятся (структура таблиц), .docx/.pdf — извлекается текст (превью ~1000 символов)
- Аудиофайлы — только метаданные (имя, размер)
- Описание передаётся в GPT для анализа

### Кнопка «Сохранить результат»
- Проверка НЕ сохраняется автоматически — студент должен нажать «Сохранить результат»
- Это позволяет перепроверять работу без траты попыток (максимум 3)

### Авторизация преподавателя
- Простая клиентская авторизация: логин `admin 1029`, пароль `hsevkrch12`
- Данные загружаются только после успешного входа

### Промпт GPT
- Системный промпт основан на методических рекомендациях ОП ИК НИУ ВШЭ
- «Кодировочный лист» и «Кодировочная таблица» указаны как синонимы
- Проверка умеренно строгая: отклонение только при явном отсутствии
